# CI/CD Integration

PipelineProbe is designed to run seamlessly in your continuous integration and deployment pipelines. By failing your build when regressions are identified, PipelineProbe acts as a quality gate for your data platform.

## Setting up GitHub Actions

We provide an example GitHub Actions workflow at `examples/github-actions/pipelineprobe.yml`. Below is the core logic:

```yaml
      - name: Run PipelineProbe Audit
        env:
          PIPELINEPROBE_AIRFLOW_PASSWORD: ${{ secrets.AIRFLOW_PASSWORD }}
          PIPELINEPROBE_WAREHOUSE_DSN: ${{ secrets.WAREHOUSE_DSN }}
        run: |
          pipelineprobe audit --config pipelineprobe.yml
```

### Authentication and Secrets

Always use your CI provider's secret management (e.g., GitHub Secrets) to pass sensitive information to PipelineProbe via environment variables:

- `PIPELINEPROBE_AIRFLOW_PASSWORD`
- `PIPELINEPROBE_WAREHOUSE_DSN`
- `PIPELINEPROBE_DBT_TARGET` (if configuring dynamically)

## Failing the Build (`fail_on_critical`)

To ensure your pipelines don't degrade over time, you can configure PipelineProbe to return a non-zero exit code if it finds too many critical issues.

In your `pipelineprobe.yml`:

```yaml
report:
  ...
  fail_on_critical: 0
```

Setting `fail_on_critical: 0` means the audit will fail the CI job if even *one* critical error is detected. A value like `5` means the CI job will pass as long as there are 5 or fewer critical errors. This is useful for adopting PipelineProbe in legacy codebases where you want to slowly burn down existing tech debt without entirely blocking developers initially.

## Archiving Reports

We highly recommend archiving the `report.json` and `pipelineprobe-report.html` files generated by PipelineProbe in your CI pipeline.

```yaml
      - name: Upload Report Artifacts
        if: always() 
        uses: actions/upload-artifact@v3
        with:
          name: pipelineprobe-reports
          path: reports/
```

This allows you to link to the HTML report in PR comments or download the JSON to track historical metrics.
